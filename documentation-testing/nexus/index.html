Identification

MX's platform is made up of dozens of distributed services, each contributing part of our solution to the complex problem of aggregating, enhancing, and delivering financial data to both end users and partner organizations.

The platform includes many purely internal tools and services, as well as many products made for partners such as RESTful web APIs, embeddable browser-based digital money management tools, cross-platform mobile money management apps, powerful customer analytics and marketing software, and much more.

  1. Creating a user (this is technically done using the MDX Real Time API);
  2. Getting an API token for that user (this is technically done using the MX SSO API);
  3. Getting a session token for that user;
  4. Creating a member using the correct institution code and credentials required by that institution;
  5. Polling the member's connection_status and answering MFA if necessary (a member aggregates automatically when it is created);
  6. Reading the member's account_owners to check whether standard aggregation returned this information (you may begin with this step for members that already exist);
  7. Calling the identify endpoint if needed;
  8. Repeating steps 5 and 6.

One reason for this workflow is that account ownership information is sometimes available through normal aggregation; in these cases, there would be no need to incur the cost of calling the identify endpoint.

For important information on errors, standards and conventions, and resource structure, please see the main documentation for the Nexus API.

1. Create a user

Use the MDX Real Time API to create a user object on MX systems.

Further details.

Example request:
Copy
  1. //Real time base URL: https://int-live.moneydesktop.com/:client_id
  2. // Endpoint: POST /users
  3. curl -i -X POST https://int-live.moneydesktop.com/:client_id/users.json \
  4. -H "Content-Type: application/vnd.moneydesktop.mdx.v5+json" \
  5. -H "Accept: application/vnd.moneydesktop.mdx.v5+json" \
  6. -H "MD-API-KEY: :api_key" \
  7. -d '{
  8. "user": {
  9. "birthdate": "1959-07-17",
  10. "credit_score": "718",
  11. "email": "walterwhite@example.com",
  12. "first_name": "Walter",
  13. "gender": "MALE",
  14. "id": "U-39XBF7",
  15. "last_name": "White",
  16. "metadata": "Additional Information",
  17. "phone": "5055551234",
  18. "zip_code": "87101"
  19. }
  20. }'

2. Get an API token

Next, you'll use the SSO API to the an api_token. This token will be used to open a Nexus session in step 3.

Remember that the user_id passed in this request must match the id specified when creating a the user in the previous step.

Each api_token is one-time use and expires in ten minutes. A fresh api_token must be requested each time a Nexus session is opened.

Further details.

Example request:
Copy
  1. SSO base URL: https://int-sso.moneydesktop.com/:client_id
  2. Endpoint: GET /users/:id/api_token
  3. curl -i https://int-sso.moneydesktop.com/:client_id/users/:user_id/api_token.xml \
  4. -H "Accept: application/vnd.moneydesktop.sso.v3+xml" \
  5. -H "MD-API-KEY: :api_key"\
Example response:
Copy
  1. {
  2. "api_token": {
  3. "token": "XXXXX"
  4. }
  5. }
