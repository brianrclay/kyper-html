Identity

Identity/account ownership information may be returned through normal aggregation. You may check whether this information is already available by calling the account owners endpoint. When such information is not returned, the identify end point may be used to gather this information on certain institutions, and then accessed with another call to the account owners endpoint.

Partners must have this premium feature enabled by MX. If this feature is not enabled, a 403 Forbidden error will be returned.

If an institution does not support identification, a 400 Bad Request error will be returned with the message, Member's institution does not support instant account identification.

Each successful identification will incur a charge. To test your identification setup, MX has provided test endpoints.

Read account owners

Use this endpoint to read any identity/account ownership data available for a specific member.

Example request:
Copy
  1. // Endpoint: GET /members/:member_guid/account_owners
  2. curl -i https://int-data.moneydesktop.com/members/MBR-c6536f4c-61dc-a047-5c45-ee38b8ba812a/account_owners \
  3. -H 'MD-SESSION-TOKEN: :session_token' \
  4. -H 'Content-Type: application/vnd.mx.nexus.v1+json' \
  5. -H 'Accept: application/vnd.mx.nexus.v1+json'

Identify

The identification workflow is similar to standard aggregation: Start the identification, poll the member, answer MFA if necessary by using the list member MFA challenges endpoint and the update member endpoint, poll the status again, then read the member's account_owners.

Before making a request to this endpoint, partners must make sure identification is enabled for the member's institution. Specifically, the institution's supports_account_identification field must be true.

If the institution does not support identification, a 400 Bad Request error will be returned with the message, Member's institution does not support instant account identification.

If a request to this endpoint fails with a status of 403 Forbidden but the member's institution nevertheless shows that identification is enabled, please contact support.

  4. -H "Content-Type: application/vnd.moneydesktop.mdx.v5+json" \
  5. -H "Accept: application/vnd.moneydesktop.mdx.v5+json" \
  6. -H "MD-API-KEY: :api_key" \
  7. -d '{

  9. "birthdate": "1959-07-17",

  15. "last_name": "White",
